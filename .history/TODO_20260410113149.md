### 4. APIサーバーの作成 (Node.js + Express)
- [ ] API用のディレクトリを作成する
  ```bash
  sudo mkdir -p /srv/http/shouchan-api
  sudo chown $USER:$USER /srv/http/shouchan-api
  cd /srv/http/shouchan-api
  npm init -y
  npm install express cors
  ```
- [ ] `server.js` を作成する（以下の内容）

  ```js
  const express = require('express')
  const cors = require('cors')
  const path = require('path')
  const fs = require('fs')

  const app = express()
  app.use(cors())
  app.use(express.json())

  // ModPackファイルを静的配信
  app.use('/modpack/files/download', express.static(path.join(__dirname, 'modpack')))

  // GET /status - サーバー死活確認
  app.get('/status', (req, res) => {
    res.json({ status: 'ok' })
  })

  // GET /modpack/info - ModPack情報
  app.get('/modpack/info', (req, res) => {
    const info = JSON.parse(fs.readFileSync('./modpack-info.json', 'utf-8'))
    res.json(info)
  })

  // GET /modpack/files - ファイル一覧
  app.get('/modpack/files', (req, res) => {
    const files = JSON.parse(fs.readFileSync('./modpack-files.json', 'utf-8'))
    res.setHeader('x-modpack-version', files.version)
    res.json(files.files)
  })

  // GET /news - ニュース
  app.get('/news', (req, res) => {
    const news = JSON.parse(fs.readFileSync('./news.json', 'utf-8'))
    res.json(news)
  })

  app.listen(3000, () => console.log('API running on port 3000'))
  ```

- [ ] `modpack-info.json` を作成する
  ```json
  {
    "version": "v1.0.0",

    "forgeVersion": "47.2.0",
  ```

- [ ] `news.json` を作成する

- [ ] `modpack-files.json` のフォーマット（後でModPackを決めてから記入）

- [ ] APIサーバーをsystemdサービスとして登録する
  ```bash
  sudo nano /etc/systemd/system/shouchan-api.service
  ```
  内容:
  ```ini

  Description=Shouchan API Server
  After=network.target

  [Service]
  Type=simple
  User=あなたのユーザー名
  WorkingDirectory=/srv/http/shouchan-api
  ExecStart=/usr/bin/node server.js
  Restart=on-failure

  [Install]
  WantedBy=multi-user.target
  ```
  ```bash
  sudo systemctl enable shouchan-api
  sudo systemctl start shouchan-api
  ```

### 5. nginxの設定
- [ ] nginx設定ファイルを編集する
  ```bash
  sudo nano /etc/nginx/sites-available/shouchan
  ```
  内容:
  ```nginx
  server {
      listen 80;
      server_name srgk.ddns.net;
      return 301 https://$host$request_uri;
  }

  server {
      listen 443 ssl;
      server_name srgk.ddns.net;

      ssl_certificate /etc/letsencrypt/live/srgk.ddns.net/fullchain.pem;
      ssl_certificate_key /etc/letsencrypt/live/srgk.ddns.net/privkey.pem;

      # ModPack静的ファイル配信
      location /modpack/files/download/ {
          alias /srv/http/shouchan-api/modpack/;
          autoindex off;
      }

      # API プロキシ
      location / {
          proxy_pass http://127.0.0.1:3000;
          proxy_http_version 1.1;
          proxy_set_header Host $host;
          proxy_set_header X-Real-IP $remote_addr;
      }
  }
  ```
- [ ] シンボリックリンクを作成する
  ```bash
  sudo ln -s /etc/nginx/sites-available/shouchan /etc/nginx/sites-enabled/
  ```
  ※ Arch Linuxはデフォルトで `sites-available/sites-enabled` がない場合がある。  
  その場合は `/etc/nginx/nginx.conf` の `http {}` ブロック内に直接 `include /etc/nginx/conf.d/*.conf;` を追加して `/etc/nginx/conf.d/shouchan.conf` に置く。

- [ ] nginx設定をテストして再起動する
  ```bash
  sudo nginx -t
  sudo systemctl reload nginx
  ```
